Basic Tree Terminology

โหนด (Node) | Node

องค์ประกอบพื้นฐานของต้นไม้ มีข้อมูลหรือค่า | Basic element of a tree, contains data or value

R

ตัวอย่าง: R, A, B, C, A1, A2 | Examples: R, A, B, C, A1, A2

ขอบ (Edge) | Edge

เส้นเชื่อมระหว่างโหนดสองตัว แสดงความสัมพันธ์แม่-ลูก | Line connecting two nodes, shows parent-child relationship

R — A

เชื่อม R กับ A | Connects R to A

ราก (Root) | Root

โหนดบนสุดของต้นไม้ ไม่มีโหนดแม่ | Top node of tree, has no parent

R

ในตัวอย่างของเรา: R | In our example: R

คุณสมบัติของต้นไม้

Tree Properties

คุณสมบัติพื้นฐาน | Basic Properties
  • เชื่อมต่อกัน (ไปถึงกันได้) | Connected (reachable)
  • ไม่มีวงจร | No cycles
  • มีเส้นทางเฉพาะระหว่างโหนดสองตัว | Unique path between any two nodes
  • n โหนด มี n-1 ขอบ | n nodes have n-1 edges

ต้นไม้ย่อย (Subtree) | Subtree

ส่วนของต้นไม้ที่เริ่มจากโหนดใดโหนดหนึ่งและรวมลูกหลานทั้งหมด | Part of tree starting from a node and including all descendants

T₁ (subtree rooted at A): A / \ A1 A2 / \ A1a A1b
สูตรสำคัญ | Important Formula

edges(T) = nodes(T) - 1

ป่า (Forest) | Forest

เซตของต้นไม้ที่แยกจากกัน | Collection of disjoint trees

Forest with 3 trees: Tree 1: A Tree 2: C Tree 3: E / \ | | B D F G / \ H I
edges(Forest) = nodes(Forest) - k
โดยที่ k = จำนวนต้นไม้ | where k = number of trees

ตรวจสอบคุณสมบัติ | Verify Properties

ตัวอย่างเป็นรูปธรรม: การตั้งปัญหา

Concrete Example: Problem Setup

เราจะพิสูจน์ว่า edges(T) = nodes(T) - 1 โดยใช้ตัวอย่างจริง | We'll prove edges(T) = nodes(T) - 1 using a real example

ต้นไม้ตัวอย่างของเรา | Our Example Tree

R (root, k=3 children) / | \ A B C /| | |\ A1 A2 B1 C1 C2 /| | A1a A1b C1a |T₁| = 5 nodes |T₂| = 2 nodes |T₃| = 4 nodes (subtree at A) (subtree at B) (subtree at C)
ข้อมูลที่กำหนด | Given Information
  • ราก R มีลูก k = 3 ตัว | Root R has k = 3 children
  • nodes(T₁) = 5 (ต้นไม้ย่อยที่ A) | (subtree at A)
  • nodes(T₂) = 2 (ต้นไม้ย่อยที่ B) | (subtree at B)
  • nodes(T₃) = 4 (ต้นไม้ย่อยที่ C) | (subtree at C)
สมมติฐานเหนี่ยวนำ | Inductive Hypothesis

สมมติว่าสูตรเป็นจริงสำหรับต้นไม้ย่อยทั้งหมด | Assume the formula holds for all subtrees:

  • edges(T₁) = nodes(T₁) - 1 = 5 - 1 = 4
  • edges(T₂) = nodes(T₂) - 1 = 2 - 1 = 1
  • edges(T₃) = nodes(T₃) - 1 = 4 - 1 = 3
🎯 เป้าหมาย: พิสูจน์ว่าสูตรเป็นจริงสำหรับต้นไม้ทั้งต้น T | Goal: Prove the formula holds for the entire tree T

การคำนวณ: นับขอบ

Calculation: Counting Edges

สูตรการนับขอบ | Edge Counting Formula
edges(T) = (Σ edges(Tᵢ)) + k
ขอบในต้นไม้ย่อย + ขอบจากรากไปลูก | Edges in subtrees + edges from root to children
ขั้นตอนที่ 1: นับขอบในต้นไม้ย่อย | Step 1: Count edges in subtrees

Σ edges(Tᵢ) = edges(T₁) + edges(T₂) + edges(T₃)

= 4 + 1 + 3 = 8

ขั้นตอนที่ 2: นับขอบจากราก | Step 2: Count edges from root

ราก R เชื่อมกับลูก 3 ตัว | Root R connects to 3 children

k = 3 edges

R ---- A (edge 1) R ---- B (edge 2) R ---- C (edge 3)
ขั้นตอนที่ 3: รวมขอบทั้งหมด | Step 3: Total all edges
edges(T) = (Σ edges(Tᵢ)) + k
= 8 + 3
= 11

การตรวจสอบ | Verification

นับขอบโดยตรง | Direct edge count:

  • R-A, R-B, R-C: 3
  • A-A1, A-A2: 2
  • A1-A1a, A1-A1b: 2
  • B-B1: 1
  • C-C1, C-C2: 2
  • C1-C1a: 1

รวม | Total: 3+2+2+1+2+1 = 11 ✓

การคำนวณ: นับโหนด

Calculation: Counting Nodes

สูตรการนับโหนด | Node Counting Formula
nodes(T) = 1 + Σ nodes(Tᵢ)
ราก + โหนดในต้นไม้ย่อย | Root + nodes in subtrees
ขั้นตอนที่ 1: นับโหนดในต้นไม้ย่อย | Step 1: Count nodes in subtrees

Σ nodes(Tᵢ) = nodes(T₁) + nodes(T₂) + nodes(T₃)

= 5 + 2 + 4 = 11

T₁: A, A1, A2, A1a, A1b (5 nodes) T₂: B, B1 (2 nodes) T₃: C, C1, C2, C1a (4 nodes)
ขั้นตอนที่ 2: เพิ่มราก | Step 2: Add the root
nodes(T) = 1 + Σ nodes(Tᵢ)
= 1 + 11
= 12

การตรวจสอบ | Verification

นับโหนดโดยตรง | Direct node count:

R A B C A1 A2 B1 C1 C2 A1a A1b C1a

รวม | Total: 12 โหนด ✓

การตรวจสอบสุดท้าย

Final Verification

การพิสูจน์เสร็จสิ้น | Proof Complete

เราได้แสดงว่า | We have shown that:

edges(T) = nodes(T) - 1

11 = 12 - 1

11 = 11 ✓

สรุปการคำนวณ | Calculation Summary

ขอบ | Edges:

edges(T) = (4 + 1 + 3) + 3 = 11

โหนด | Nodes:

nodes(T) = 1 + (5 + 2 + 4) = 12

ผลลัพธ์ | Result:

edges(T) = nodes(T) - 1 ✓

ขั้นตอนการพิสูจน์เหนี่ยวนำ | Inductive Proof Steps

  1. พื้นฐาน | Base: สูตรเป็นจริงสำหรับต้นไม้ย่อย | Formula true for subtrees
  2. ขั้นตอน | Step: แสดงสำหรับต้นไม้ทั้งต้น | Show for entire tree
  3. สรุป | Conclusion: edges = nodes - 1
🎯 ข้อสำคัญ: การพิสูจน์เหนี่ยวนำเชิงโครงสร้างใช้โครงสร้างของข้อมูล (ต้นไม้) เป็นแนวทางในการพิสูจน์ เราสมมติว่าคุณสมบัติเป็นจริงสำหรับชิ้นส่วนที่เล็กกว่า (ต้นไม้ย่อย) แล้วแสดงว่าเป็นจริงสำหรับโครงสร้างทั้งหมด | Key Insight: Structural induction uses the structure of the data (tree) as a guide for proof. We assume the property holds for smaller pieces (subtrees) and show it holds for the entire structure

Summary

🌳 ศัพท์เฉพาะที่เรียนรู้ | Terminology Learned

  • โหนด (Node): องค์ประกอบพื้นฐาน | Node: Basic element
  • ขอบ (Edge): เส้นเชื่อม | Edge: Connection line
  • ราก (Root): โหนดบนสุด | Root: Top node
  • ใบ (Leaf): โหนดไม่มีลูก | Leaf: Node with no children
  • ต้นไม้ย่อย (Subtree): ส่วนของต้นไม้ | Subtree: Part of tree
  • ป่า (Forest): หลายต้นไม้ | Forest: Multiple trees

📊 การพิสูจน์เหนี่ยวนำเชิงโครงสร้าง | Structural Induction

  • ใช้โครงสร้างข้อมูลเป็นแนวทาง | Uses data structure as guide
  • สมมติเป็นจริงสำหรับส่วนย่อย | Assume true for subparts
  • แสดงสำหรับโครงสร้างทั้งหมด | Show for entire structure
  • เหมาะสำหรับการพิสูจน์คุณสมบัติต้นไม้ | Perfect for tree properties

🔢 สูตรสำคัญที่พิสูจน์แล้ว | Important Formula Proven

edges(T) = nodes(T) - 1

สำหรับต้นไม้ใดๆ ที่เชื่อมต่อและไม่มีวงจร | For any connected, acyclic tree

💡 ข้อควรจำ | Key Takeaways

  • ต้นไม้มีโครงสร้างแบบเรียกซ้ำ | Trees have recursive structure
  • การพิสูจน์เหนี่ยวนำสะท้อนโครงสร้างนี้ | Induction mirrors this structure
  • ตัวอย่างเป็นรูปธรรมช่วยให้เข้าใจ | Concrete examples aid understanding
  • การนับและการคำนวณสามารถตรวจสอบได้ | Counting and calculations are verifiable

🚀 การประยุกต์ใช้ | Applications

  • อัลกอริทึมบนต้นไม้ | Tree algorithms
  • โครงสร้างข้อมูล | Data structures
  • การวิเคราะห์ความซับซ้อน | Complexity analysis
  • การออกแบบฐานข้อมูล | Database design
